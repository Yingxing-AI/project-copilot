# v0.3.2

## Changes since v0.3.1

- Added ADR 0008 to define the Validation Snapshot lifecycle.
- Clarified that `.ai/validation.json` is a derived, overwriteable snapshot rather than a long-term memory source.
- Clarified that `.ai/derived/metrics.json` must refresh together with `validation.json`.
- Clarified that `docs/validation-report.md` is a rebuildable view and must not become a fact source.
- Updated `close_day` so confirmed Session Archive writes now refresh Validation derivatives.
- Synced governance rules across ADR index, AGENTS, Codex workflow docs, generated Codex workflow content, and tests.
- Added regression coverage to ensure session-candidate-only updates do not refresh `validation.json`, confirmed archive writes do refresh validation artifacts, and validation reports prefer live `.ai` memory over stale snapshots.

## Stable Scope

- Project Copilot remains a local-first memory layer for Codex.
- Charter, ADR, Session Archive, Validation, Memory Health, and Governance stay as the stable core surfaces.
- Validation remains derived from real `.ai` state and does not become a primary memory source.
