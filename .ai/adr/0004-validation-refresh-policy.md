# ADR 0004: 验证报告跟随长期记忆写入自动刷新

日期：2026-06-19

状态：Accepted

背景：

Validation Report 和 Case Study 如果依赖人工维护，容易漂移。Project Copilot 已收敛为项目记忆层，验证数据应该成为真实 `.ai` 记忆的自然副产品，而不是单独维护的一套文档。

决策：

长期记忆或验证快照写入后自动刷新验证报告；Session 候选、未确认假设和普通开发流水不触发刷新。

原因：

这样可以避免用户每次手动刷新，同时符合 Session Memory 的低写入原则。候选事件尚未确认，不应进入派生验证数据。

取舍：

获得更一致的验证报告和更少的用户操作。放弃开发过程中实时刷新验证统计。

影响：

`init`、`adopt`、确认决策、同步项目状态和导出验证快照后会自动刷新。`continue`、候选事件和未确认假设不刷新。手动刷新命令只作为兼容和修复入口保留。
