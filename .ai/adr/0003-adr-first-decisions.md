# ADR 0003: 将决策记录升级为 ADR 体系

日期：2026-06-19

状态：Accepted

背景：

AI Coding 最容易遗忘的是“为什么这样做”，而不是“昨天做了什么”。单个 `DECISIONS.md` 文件随着时间增长后难以检索、引用和废弃旧决策。

决策：

将 `.ai/DECISIONS.md` 演进为 ADR 兼容索引，新决策优先写入 `.ai/adr/NNNN-title.md`。

原因：

ADR 能以小文件记录背景、决策、原因、取舍和影响，适合未来 Codex 在开发前快速理解历史约束。

取舍：

获得更强的可引用性和决策演进能力。放弃把所有决策追加到单个 Markdown 文件的长期模式。

影响：

`.ai/DECISIONS.md` 保留历史兼容摘要，不覆盖旧决策。新架构决策、MVP 范围变化、重大依赖变化和放弃已有功能应进入 `.ai/adr/`。
